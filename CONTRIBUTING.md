# 贡献指南

感谢你考虑为 OpenHands Cookbook 做贡献！🎉

---

## 🤝 如何贡献

### 方式一：提交 Issue

- 🐛 **Bug 报告** - 发现错误或不准确的信息
- 💡 **功能建议** - 想要添加新的案例或章节
- 📝 **内容改进** - 想要优化现有内容
- ❓ **问题反馈** - 遇到不清楚的地方

**步骤**：
1. 访问 [GitHub Issues](https://github.com/srxly888-creator/openhands-cookbook/issues)
2. 点击 "New Issue"
3. 选择合适的模板
4. 填写详细信息

### 方式二：提交 Pull Request

**步骤**：

1. **Fork 仓库**
   ```bash
   git clone https://github.com/srxly888-creator/openhands-cookbook.git
   cd openhands-cookbook
   ```

2. **创建分支**
   ```bash
   git checkout -b feature/your-feature-name
   ```

3. **进行修改**
   - 添加新内容
   - 修复错误
   - 优化表达

4. **提交更改**
   ```bash
   git add .
   git commit -m "feat: 添加新案例 - FastAPI 用户认证"
   ```

5. **推送到 GitHub**
   ```bash
   git push origin feature/your-feature-name
   ```

6. **创建 Pull Request**
   - 访问你的 Fork 仓库
   - 点击 "New Pull Request"
   - 填写 PR 描述

## 📝 贡献类型

### 1. 文档改进

- ✅ 修复错别字
- ✅ 优化语句表达
- ✅ 添加更多示例
- ✅ 改进格式排版

### 2. 新增案例

我们欢迎所有类型的案例！

**优先级**：
- 🔥 **高优先级** - Web 开发、数据分析、自动化
- ⭐ **中优先级** - 机器学习、DevOps、测试
- 💡 **低优先级** - 其他有趣的用例

**案例模板**：
```markdown

> [简短描述]

[详细描述任务]

\`\`\`bash
openhands run "任务描述"
\`\`\`

## 📄 预期结果

### 代码示例

\`\`\`python

\`\`\`

\`\`\`bash

\`\`\`

## 🎓 学习要点

- ✅ 要点 2
- ✅ 要点 3
```

## ✅ 代码规范

### Markdown 格式

- 使用 `#` 表示标题
- 使用 ``` 标记代码块
- 使用 `>` 表示引用
- 使用 `-` 表示列表

### 代码示例

- 提供完整的可运行代码
- 添加必要的注释
- 遵循语言的最佳实践

### 文件命名

- 使用小写字母和连字符
- 例如：`fastapi-auth.md`、`data-analysis.md`

## 🎯 贡献流程

### 1. 选择任务

- 查看 [Issues](https://github.com/srxly888-creator/openhands-cookbook/issues)
- 选择你感兴趣的 Issue
- 评论表示你想要处理

### 2. 开发和测试

- 在本地进行修改
- 确保所有链接有效
- 检查格式正确

### 3. 提交 PR

- 填写清晰的 PR 标题
- 提供详细的 PR 描述
- 关联相关的 Issue

### 4. Code Review

- 响应审查意见
- 进行必要的修改
- 等待合并
